## 2. 我们收集什么数据

**简短回答:不收集任何数据。**

详细说明:
- Judge 在你浏览器本地分析当前网页的性能
- 所有数据**仅存在于你的浏览器内存中**
- 关闭标签页或浏览器,数据立即清除
- **没有任何后端服务器**接收用户数据
- **没有任何分析 / 埋点 / 遥测**

## 3. 我们读什么数据(本地)

为了完成性能诊断,Judge 在你浏览器本地读取下列数据:

| 数据类别 | 用途 | 存储位置 |
|---|---|---|
| 当前页面的 fetch / XMLHttpRequest 请求 URL、HTTP 状态码、耗时 | 生成慢接口排名 | 浏览器内存 |
| API 响应体(最多 64 KB / 请求) | 检测业务层失败 | 浏览器内存 |
| 页面 Web Vitals(LCP / CLS / INP / FCP / TTFB) | 体验指标评分 | 浏览器内存 |
| 用户最近 30 个操作(点击 / 输入 / 跳转) | 操作轨迹回放 | 浏览器内存 |
| JS 运行时异常 + 资源加载失败 | 异常根因分析 | 浏览器内存 |
| `navigator.connection`(估算下行 / RTT) | 网络评分 | 浏览器内存 |

**特别说明 — 用户操作轨迹的隐私保护**:
- 输入字段(`<input>`)**不抓 value**,只记录元素的 selector 和输入的字符**长度**
- 类型为 `password` 的输入框完全跳过,任何信息都不记录
- 不读 cookie、不读 localStorage、不读 sessionStorage
- 不附着任何 form 表单提交事件

## 4. 数据离开你浏览器的时机

**只有当你主动点击下面任一按钮时**:

- "复制摘要" → 内容写入你的剪贴板,**你**决定粘贴到哪里
- "下载 PDF 报告" → 文件存到你电脑的下载目录,**你**决定发给谁
- "下载 JSON" → 同上

不点这些按钮 = 数据完全没离开你的浏览器内存。

## 5. 唯一的外部网络请求(可选)

为了在 PDF 报告右上角生成"采样人公网 IP"水印(给开发同学定位地区用),
Judge 会查询以下三个公网 IP API 之一:

- `https://api.ipify.org`
- `https://ipinfo.io/json`
- `https://ifconfig.co/json`

**这些请求**:
- 仅在你**实际生成 PDF 报告**时触发(不下载就不会发)
- 只返回你的公网 IP 字符串(已经是公开信息,任何网站都能看到)
- 不发送任何身份信息、cookie、扩展数据
- 失败时降级为 "—",不阻塞 PDF 生成
- 想完全离线?把扩展从开发者模式挂在断网状态下也能用,只是 PDF 没 IP 水印

## 6. 域名 ping 主动检测

可选功能(用户必须**点按钮**才会触发):
- "实测 RTT" — 对当前页面同源发 6 次 GET 小请求,测延迟
- "实测各域名" — 对页面用到的每个 API 域名发 3 次 GET,测连通性

这些请求都在用户**主动触发**后才发出,带 `cache: 'no-store'` 防缓存,
**不带 credentials**(不发 cookie / 认证信息)。

## 7. 权限说明

| 权限 | 为什么需要 |
|---|---|
| `activeTab` | 仅在用户点扩展图标时,临时访问当前标签的 Performance API |
| `scripting` | 把分析脚本注入当前页面的所有 frame,运行性能采集 |
| `storage` | 在"完整报告"功能中,从弹窗向报告页传递诊断数据(短暂的 in-extension 存储) |
| `clipboardWrite` | 实现"复制摘要"按钮 |
| `downloads` | 实现"下载 PDF / JSON"按钮 |
| `host_permissions: <all_urls>` | 用户可能在任意网站上使用 Judge,无法预先限定域名 |
| Content scripts on `<all_urls>` | 同上,需要在用户访问的页面上运行性能采集 |

**没有的权限**:
- ❌ `webRequest` / `webRequestBlocking`(不修改请求)
- ❌ `cookies`(不读 cookie)
- ❌ `history`(不读浏览历史)
- ❌ `tabs` 全权限(只用 activeTab)
