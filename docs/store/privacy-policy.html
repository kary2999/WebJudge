2. 我们收集什么数据

简短回答:不收集任何数据。

  • Judge 在你浏览器本地分析当前网页的性能
  • 所有数据仅存在于你的浏览器内存中
  • 关闭标签页或浏览器,数据立即清除
  • 没有任何后端服务器接收用户数据
  • 没有任何分析 / 埋点 / 遥测

3. 我们读什么数据(本地,仅在浏览器内存)

数据类别用途存储位置
当前页面的 fetch / XMLHttpRequest 请求 URL、HTTP 状态、耗时生成慢接口排名浏览器内存
API 响应体(最多 64 KB / 请求)检测业务层失败浏览器内存
页面 Web Vitals (LCP / CLS / INP / FCP / TTFB)体验指标评分浏览器内存
用户最近 30 个操作(点击 / 输入 / 跳转)操作轨迹回放浏览器内存
JS 运行时异常 + 资源加载失败异常根因分析浏览器内存
navigator.connection(估算下行 / RTT)网络评分浏览器内存
用户操作轨迹的隐私保护(关键):
  • 输入字段(<input>)不抓 value,只记录元素的 selector 和输入的字符长度
  • 类型为 password 的输入框完全跳过,任何信息都不记录
  • 不读 cookie、不读 localStorage、不读 sessionStorage
  • 不附着任何 form 表单提交事件

4. 数据离开你浏览器的时机

只有当你主动点击下面任一按钮时:

  • "复制摘要" → 内容写入你的剪贴板,你决定粘贴到哪里
  • "下载 PDF 报告" → 文件存到你电脑的下载目录,你决定发给谁
  • "下载 JSON" → 同上

不点这些按钮 = 数据完全没离开你的浏览器内存。

5. 唯一的外部网络请求(可选)

为了在 PDF 报告右上角生成"采样人公网 IP"水印,Judge 会查询以下三个公网 IP API 之一:

  • https://api.ipify.org
  • https://ipinfo.io/json
  • https://ifconfig.co/json

这些请求:

  • 仅在你实际生成 PDF 报告时触发
  • 只返回你的公网 IP 字符串(已经是公开信息,任何网站都能看到)
  • 不发送任何身份信息、cookie、扩展数据
  • 失败时降级为 "—",不阻塞 PDF 生成

6. 域名 ping 主动检测

可选功能(用户必须点按钮才会触发):

  • "实测 RTT" — 对当前页面同源发 6 次 GET 小请求,测延迟
  • "实测各域名" — 对页面用到的每个 API 域名发 3 次 GET,测连通性

这些请求都在用户主动触发后才发出,带 cache: 'no-store' 防缓存,不带 credentials(不发 cookie / 认证信息)。

7. 权限说明

权限为什么需要
activeTab仅在用户点扩展图标时,临时访问当前标签的 Performance API
scripting把分析脚本注入当前页面的所有 frame,运行性能采集
storage从弹窗向报告页传递诊断数据(短暂的 in-extension 存储,1 小时自动清理)
clipboardWrite实现"复制摘要"按钮
downloads实现"下载 PDF / JSON"按钮
host_permissions: <all_urls>用户可能在任意网站上使用 Judge,无法预先限定域名

没有的权限:

  • ❌ webRequest / webRequestBlocking(不修改请求)
  • ❌ cookies(不读 cookie)
  • ❌ history(不读浏览历史)
  • ❌ tabs 全权限(只用 activeTab)
